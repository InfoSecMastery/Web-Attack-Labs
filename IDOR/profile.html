🎓 IDOR Vulnerability Demonstrated Here

What is IDOR? Insecure Direct Object Reference occurs when an application exposes a reference to an internal implementation object (like a user ID, database key, or filename) in a URL or parameter, without proper authorization checks.

How to exploit: After logging in, notice the profile URL includes ?user_id=X. If you change X to another user's ID (1, 2, or 3), you can view their full profile — including balance, address, phone number, and other sensitive data — without any authorization check.

The Fix: A secure application would verify that the logged-in user's ID matches the requested profile ID before returning any data. This demo intentionally omits that check to teach the vulnerability.
